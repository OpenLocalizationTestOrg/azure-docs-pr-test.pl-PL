# <a name="data-catalog-developer-samples"></a>Przykłady dewelopera wykazu danych
Rozpocznij tworzenie wykazu danych aplikacji przy użyciu interfejsu API REST wykazu danych. Interfejs API REST wykazu danych jest opartego na interfejsie REST API, która zapewnia dostęp programistyczny do wykazu danych zasobów w celu rejestracji, dodawać adnotacje i programowo wyszukiwania zasobów danych.

Poniżej przedstawiono przykłady, który mamy w witrynie GitHub:

* [Rozpoczynanie pracy z usługą Azure Data Catalog](https://azure.microsoft.com/resources/samples/data-catalog-dotnet-get-started/)
  
  Przykładowe który przedstawiono sposób uwierzytelniania za pomocą usługi Azure AD, aby zarejestrować, wyszukiwanie i usuwanie zasobu danych przy użyciu interfejsu API REST wykazu danych.
* [Zbiorcze rejestrowanie i adnotacji](https://azure.microsoft.com/resources/samples/data-catalog-dotnet-excel-register-data-assets/)
  
  Przykład pokazujący, jak przeprowadzić zbiorczą rejestrować zasoby danych ze skoroszytu programu Excel przy użyciu interfejsu API REST katalogu danych i Open XML.
* [Narzędzie importu/eksportu](https://azure.microsoft.com/resources/samples/data-catalog-dotnet-import-export/)
  
  Przykład pokazujący sposób pobrać zasoby z usługą Azure Data Catalog i serializacji je do pliku za pomocą interfejsu API REST wykazu danych. On również pokazano, jak pobrać zestaw zasobów zserializowanym w formacie JSON i wypychanie ich do katalogu. Obsługuje ona eksportowanie podzbiór katalogu przy użyciu zapytania wyszukiwania.

* [Zbiorcze importu terminów](https://azure.microsoft.com/resources/samples/data-catalog-bulk-import-glossary/)

    W tym przykładzie pokazano, jak zaimportować terminów z plików CSV do ADC słownik.

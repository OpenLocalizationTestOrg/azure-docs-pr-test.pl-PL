## <a name="how-it-works"></a><span data-ttu-id="0769c-113">Jak to działa</span><span class="sxs-lookup"><span data-stu-id="0769c-113">How it works</span></span>
![Omówienie usługi CDN](./media/cdn-overview/cdn-overview.png)

1. <span data-ttu-id="0769c-115">Użytkownik (Anna) żąda pliku (nazywanego również zasobem) przy użyciu adresu URL ze specjalną nazwą domeny, taką jak `<endpointname>.azureedge.net`.</span><span class="sxs-lookup"><span data-stu-id="0769c-115">A user (Alice) requests a file (also called an asset) using a URL with a special domain name, such as `<endpointname>.azureedge.net`.</span></span>  <span data-ttu-id="0769c-116">System DNS kieruje żądanie do najlepiej działającej lokalizacji punktu obecności (POP, Point-of-Presence).</span><span class="sxs-lookup"><span data-stu-id="0769c-116">DNS routes the request to the best performing Point-of-Presence (POP) location.</span></span>  <span data-ttu-id="0769c-117">Zazwyczaj jest to punkt obecności znajdujący się geograficznie najbliżej użytkownika.</span><span class="sxs-lookup"><span data-stu-id="0769c-117">Usually this is the POP that is geographically closest to the user.</span></span>
2. <span data-ttu-id="0769c-118">Jeśli serwery krawędzi w lokalizacji POP nie mają pliku w swojej pamięci podręcznej, serwer krawędzi żąda pliku z punktu początkowego.</span><span class="sxs-lookup"><span data-stu-id="0769c-118">If the edge servers in the POP do not have the file in their cache, the edge server requests the file from the origin.</span></span>  <span data-ttu-id="0769c-119">Punktem początkowym może być aplikacja sieci Web platformy Azure, usługa Azure Cloud Service, konto usługi Azure Storage lub dowolny publicznie dostępny serwer sieci Web.</span><span class="sxs-lookup"><span data-stu-id="0769c-119">The origin can be an Azure Web App, Azure Cloud Service, Azure Storage account, or any publicly accessible web server.</span></span>
3. <span data-ttu-id="0769c-120">Punkt początkowy zwraca plik do serwera krawędzi, dołączając opcjonalne nagłówki HTTP opisujące czas wygaśnięcia (TTL, Time-to-Live) pliku.</span><span class="sxs-lookup"><span data-stu-id="0769c-120">The origin returns the file to the edge server, including optional HTTP headers describing the file's Time-to-Live (TTL).</span></span>
4. <span data-ttu-id="0769c-121">Serwer krawędzi zapisuje plik w pamięci podręcznej i zwraca go do oryginalnego obiektu żądającego (Anna).</span><span class="sxs-lookup"><span data-stu-id="0769c-121">The edge server caches the file and returns the file to the original requestor (Alice).</span></span>  <span data-ttu-id="0769c-122">Plik pozostaje w pamięci podręcznej na serwerze krawędzi do momentu upłynięcia czasu wygaśnięcia.</span><span class="sxs-lookup"><span data-stu-id="0769c-122">The file remains cached on the edge server until the TTL expires.</span></span>  <span data-ttu-id="0769c-123">Jeśli punkt początkowy nie określił czasu wygaśnięcia, domyślnie wynosi on siedem dni.</span><span class="sxs-lookup"><span data-stu-id="0769c-123">If the origin didn't specify a TTL, the default TTL is seven days.</span></span>
5. <span data-ttu-id="0769c-124">Dodatkowi użytkownicy mogą następnie zażądać tego samego pliku przy użyciu tego samego adresu URL, a także mogą być kierowani do tego samego punktu obecności.</span><span class="sxs-lookup"><span data-stu-id="0769c-124">Additional users may then request the same file using that same URL, and may also be directed to that same POP.</span></span>
6. <span data-ttu-id="0769c-125">Jeśli czas wygaśnięcia pliku nie upłynął, serwer krawędzi zwraca plik z pamięci podręcznej.</span><span class="sxs-lookup"><span data-stu-id="0769c-125">If the TTL for the file hasn't expired, the edge server returns the file from the cache.</span></span>  <span data-ttu-id="0769c-126">Skutkuje to szybszymi czasami reakcji w środowisku użytkownika.</span><span class="sxs-lookup"><span data-stu-id="0769c-126">This results in a faster, more responsive user experience.</span></span>

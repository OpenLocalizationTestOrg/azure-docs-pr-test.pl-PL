# <a name="service-request-sr-information"></a>Informacje o żądaniu (SR) usługi
Podczas otwierania żądania obsługi dla żadnej usługi Mobile Engagement, należy podać następujące informacje:

## <a name="ids-applicable-identifiers-related-to-your-issue"></a>Identyfikatorów: Identyfikatory odpowiednich powiązanym z problemem:
* Identyfikator aplikacji
* Identyfikator kampanii
* Identyfikator urządzenia
* Identyfikator użytkownika
* Nazwa użytkownika
* Tag informacje o aplikacji

## <a name="errors-applicable-error-information"></a>Błędy: Informacje o błędzie dotyczy
* Nazwa sekcji interfejsu API lub interfejsu użytkownika, w której występuje dany problem
* Tekst wszelkie komunikaty o błędach, które otrzymujesz
* Wyniki testów wykonanych z [przewodniki rozwiązywania problemów](http://go.microsoft.com/fwlink/?LinkId=524382)

## <a name="code-applicable-coding-information"></a>Kodu: Odpowiednie informacje kodowania
* Wersja zestawu SDK i platform aplikacji (zestaw SDK systemu Android 2.4.1, iOS 1.16.2 itp.)
* Pobieranie lokalizacji aplikacji produkcyjnych (lub plik APK/TGZ rozwoju aplikacji)
* "AndroidManifest.xml" i/lub dowolnego fragment kodu z aplikacji związane z usługi Azure Mobile Engagement (w przypadku zaawansowanego rozwiązywania problemów)

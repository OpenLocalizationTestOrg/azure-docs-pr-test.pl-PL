## <a name="example"></a><span data-ttu-id="36f22-115">Przykład</span><span class="sxs-lookup"><span data-stu-id="36f22-115">Example</span></span>
<span data-ttu-id="36f22-116">Poniższy kod przedstawia użycie wielu niezależnych zapytań za pomocą danego **ShardMap** o nazwie *myShardMap*.</span><span class="sxs-lookup"><span data-stu-id="36f22-116">The following code illustrates the usage of multi-shard querying using a given **ShardMap** named *myShardMap*.</span></span> 

    using (MultiShardConnection conn = new MultiShardConnection( 
                                        myShardMap.GetShards(), 
                                        myShardConnectionString) 
          ) 
    { 
    using (MultiShardCommand cmd = conn.CreateCommand())
           { 
            cmd.CommandText = "SELECT c1, c2, c3 FROM ShardedTable"; 
            cmd.CommandType = CommandType.Text; 
            cmd.ExecutionOptions = MultiShardExecutionOptions.IncludeShardNameColumn; 
            cmd.ExecutionPolicy = MultiShardExecutionPolicy.PartialResults; 

            using (MultiShardDataReader sdr = cmd.ExecuteReader()) 
                { 
                    while (sdr.Read())
                        { 
                            var c1Field = sdr.GetString(0); 
                            var c2Field = sdr.GetFieldValue<int>(1); 
                            var c3Field = sdr.GetFieldValue<Int64>(2);
                        } 
                 } 
           } 
    } 


<span data-ttu-id="36f22-117">Najważniejsza różnica polega na konstrukcji niezależnych wielu połączeń.</span><span class="sxs-lookup"><span data-stu-id="36f22-117">A key difference is the construction of multi-shard connections.</span></span> <span data-ttu-id="36f22-118">Gdzie **SqlConnection** działa na pojedynczej bazy danych **MultiShardConnection** przyjmuje ***kolekcji odłamków*** jako dane wejściowe.</span><span class="sxs-lookup"><span data-stu-id="36f22-118">Where **SqlConnection** operates on a single database, the **MultiShardConnection** takes a ***collection of shards*** as its input.</span></span> <span data-ttu-id="36f22-119">Wypełnianie kolekcji odłamków z mapy niezależnego fragmentu.</span><span class="sxs-lookup"><span data-stu-id="36f22-119">Populate the collection of shards from a shard map.</span></span> <span data-ttu-id="36f22-120">Następnie wykonać zapytania w kolekcji przy użyciu fragmentów **UNION ALL** semantyki można złożyć jeden wynik ogólny.</span><span class="sxs-lookup"><span data-stu-id="36f22-120">The query is then executed on the collection of shards using **UNION ALL** semantics to assemble a single overall result.</span></span> <span data-ttu-id="36f22-121">Opcjonalnie można dodać nazwę niezależnego fragmentu, z której pochodzi wiersz do danych wyjściowych przy użyciu **ExecutionOptions** właściwość polecenia.</span><span class="sxs-lookup"><span data-stu-id="36f22-121">Optionally, the name of the shard where the row originates from can be added to the output using the **ExecutionOptions** property on command.</span></span>

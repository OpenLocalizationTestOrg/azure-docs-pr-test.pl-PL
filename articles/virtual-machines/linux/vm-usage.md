## <a name="service-type"></a>Typ usługi
Pole typu usługi, w polu informacje dodatkowe odpowiada dokładnie rozmiar maszyny Wirtualnej wdrożonej. Magazyn w warstwie Premium maszyn wirtualnych (opartych na dysk SSD) i inne niż premium magazynu maszyn wirtualnych (opartych na dysk twardy) są takie same cenach. W przypadku wdrożenia na dyskach SSD rozmiar, takich jak standardowy\_DS2\_v2, zostanie wyświetlony rozmiar z systemem innym niż dysk SSD ("standardowe\_D2\_v2 maszyny Wirtualnej") w kolumnie podkategorii miernika i rozmiaru dysków SSD ("standardowe\_DS2\_ v2 ") w polu informacje dodatkowe.

## <a name="virtual-machine-usage-faq"></a>Obciążenie maszyny wirtualnej — często zadawane pytania
### <a name="what-resources-are-charged-when-deploying-a-vm"></a>Jakie zasoby są naliczane podczas wdrażania maszyny Wirtualnej?    
Maszyny wirtualne uzyskać kosztów dla maszyny Wirtualnej, oprogramowanie premium uruchomione na maszynie Wirtualnej, dysku account\managed magazynu skojarzone z maszyną Wirtualną, oraz przepustowość sieci przesyła z maszyny Wirtualnej.
### <a name="how-can-i-tell-if-a-vm-is-using-azure-hybrid-benefit-in-the-usage-csv"></a>Jak sprawdzić, czy maszyny Wirtualnej używa Azure hybrydowego korzyści w formacie CSV użycia?
W przypadku wdrożenia przy użyciu [korzyści hybrydowego Azure](https://azure.microsoft.com/pricing/hybrid-benefit/), szybkość maszyny Wirtualnej systemu Windows nie są naliczane, ponieważ własnej licencji są wprowadzeniem do chmury. W rachunku, można rozróżnić której Menedżer zasobów maszyny wirtualne są uruchomione Azure hybrydowego korzyści, ponieważ mają one "Windows\_serwera BYOL" lub "Windows\_klienta BYOL" w kolumnie ImageType.
### <a name="how-are-basic-vs-standard-vm-types-differentiated-in-the-usage-csv"></a>Jak są podstawowe programu vs. Standardowe typy wirtualna zróżnicowana w formacie CSV użycia?
Są dostępne zarówno Basic i Standard A-Series maszyn wirtualnych. W przypadku wdrożenia podstawowej maszyny Wirtualnej w kategorii licznika Sub składa się z ciągu "Basic". W przypadku wdrożenia maszyny Wirtualnej standardowego A-Series, następnie rozmiar maszyny Wirtualnej jest wyświetlany jako "Maszyna wirtualna A1" ponieważ Standard jest ustawieniem domyślnym. Aby dowiedzieć się więcej na temat różnic między Basic i Standard, zobacz [stronie cennika](https://azure.microsoft.com/pricing/details/virtual-machines/).
### <a name="what-are-extrasmall-small-medium-large-and-extralarge-sizes"></a>Co to są rozmiary ExtraSmall, małych, średnich, duże i ExtraLarge?
ExtraSmall — ExtraLarge są starsze nazwy dla standardowej\_A0 — Standard\_A4. W klasycznym rekordów użycia maszyny Wirtualnej można napotkać tę Konwencję używane, jeśli wdrożono rozmiary.
### <a name="what-is-the-difference-between-meter-region-and-resource-location"></a>Jaka jest różnica między miernika regionu i lokalizacji zasobów?
Region licznik jest skojarzony z licznika. W przypadku niektórych usług Azure, korzystających z jednego cen dla wszystkich regionów pole Region licznika może być pusta. Jednak ponieważ maszyny wirtualne są wyposażone w dedykowane ceny dla regionu dla maszyn wirtualnych, to pole jest wypełniane. Podobnie lokalizacji zasobów dla maszyn wirtualnych jest lokalizacja, w której wdrażana jest maszyna wirtualna. Region platformy Azure w obu polach są takie same, chociaż może zawierać Konwencja inny ciąg Nazwa regionu.
### <a name="why-is-the-imagetype-value-blank-in-the-additional-info-field"></a>Dlaczego jest ImageType wartość pusta w polu informacje dodatkowe?
W polu ImageType znajduje się tylko dla podzbioru obrazów. Jeśli nie zostały wdrożone, jeden z obrazów powyżej, ImageType jest pusta.
### <a name="why-is-the-vmname-blank-in-the-additional-info"></a>VMName jest pusty w dodatkowych informacji
VMName jest tylko wypełnione pola dodatkowe informacje dla maszyn wirtualnych w zestawie skalowania. W polu Identyfikator wystąpienia zawiera nazwę maszyny Wirtualnej dla maszyn wirtualnych zestawu z systemem innym niż skalowania.
### <a name="what-does-computehr-mean-in-the-usagetype-field-in-the-additional-info"></a>Co oznacza ComputeHR w polu UsageType dodatkowych informacji?
Godziny obliczeniowe, reprezentujący zdarzenia użycia podstawowej kosztów infrasturcture oznacza ComputeHR. W przypadku UsageType ComputeHR\_SW zdarzenia użycia reprezentuje premium bezpłatnie oprogramowania dla maszyny Wirtualnej.
### <a name="how-do-i-know-if-i-am-charged-for-premium-software"></a>Jak sprawdzić, jeśli mam naliczona opłata oprogramowania — wersja premium
Podczas eksplorowania obrazu maszyny Wirtualnej, która najlepiej odpowiada potrzebom użytkownika, należy koniecznie zapoznaj się z [portalu Azure Marketplace](https://azuremarketplace.microsoft.com/marketplace/apps/category/compute). Obraz ma szybkość planu oprogramowania. Jeśli zobaczysz częstotliwość "Wolne", nie ma żadnych dodatkowych kosztów, oprogramowania. 
### <a name="what-is-the-difference-between-microsoftclassiccompute-and-microsoftcompute-in-the-consumed-service"></a>Jaka jest różnica między obszar Microsoft.ClassicCompute i Microsoft.Compute w usłudze zużyto?
Obszar Microsoft.ClassicCompute reprezentuje zasoby klasyczne wdrożone za pośrednictwem usługi Azure Service Manager. Jeśli wdrożono za pomocą Menedżera zasobów Microsoft.Compute jest wypełniana w usłudze wykorzystanych. Dowiedz się więcej o [modele wdrażania Azure](/azure-resource-manager/resource-manager-deployment-model.md).
### <a name="why-is-the-instanceid-field-blank-for-my-virtual-machine-usage"></a>W polu InstanceID jest puste w przypadku użycia maszyny wirtualnej
W przypadku wdrożenia przy użyciu klasycznego modelu wdrażania ciąg InstanceID nie jest dostępna.
### <a name="why-are-the-tags-for-my-vms-not-flowing-to-the-usage-details"></a>Dlaczego są tagi nie są przesyłane do Szczegóły obciążenia maszyn wirtualnych?
Tagi przepływać tylko dla Ciebie CSV użycia dla maszyn wirtualnych Menedżera zasobów tylko. Tagi zasobów klasycznych nie są dostępne w szczegółach użycia.
### <a name="how-can-the-consumed-quantity-be-more-than-24-hours-one-day"></a>Jak ilość wykorzystanego może być dłuższy niż 24 godziny na jeden dzień
W klasycznym modelu rozliczeń dla zasobów są agregowane na poziomie usługi w chmurze. Jeśli masz więcej niż jedną maszynę Wirtualną w usłudze w chmurze, która używa tego samego licznika rozliczeń, użycie jest agregowana razem. Maszyn wirtualnych wdrożonych za pośrednictwem Menedżera zasobów są rozliczane na poziomie maszyny Wirtualnej, więc nie będzie stosowana ta agregacja.
### <a name="why-is-pricing-not-available-for-dsfsgsls-sizes-on-the-pricing-page"></a>Dlaczego jest nie ceny na stronie cen dostępnych rozmiarów DS/FS/GS/LS?
Magazyn w warstwie Premium obsługuje maszyny wirtualne są rozliczane na tym samym poziomie jak inne niż premium magazynu obsługujących maszyny wirtualne. Różnią się tylko kosztów magazynowania. Odwiedź stronę [magazynu cennikiem](https://azure.microsoft.com/pricing/details/storage/unmanaged-disks/) Aby uzyskać więcej informacji.

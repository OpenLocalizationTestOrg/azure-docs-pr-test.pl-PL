# <a name="what-does-azure-multi-factor-authentication-mean-for-me"></a>Co oznacza Azure Multi-Factor Authentication dla mnie?
Tak, ma zaakceptujesz wiadomość e-mail od użytkowników z IT lub Twoje przełożonego, informujący o tym, ich dodanie dodatkowej weryfikacji zabezpieczeń do konta.  Dlatego co to znaczy?

Nie martw się. Organizacja chce po prostu wykonać dodatkowe czynności, aby upewnić się, za kogo się podaje, to podczas logowania się. Ta weryfikacja odbywa się przy użyciu kombinacji nazwy użytkownika i hasła i telefonu.  

Dlatego najpierw należy przeprowadzić proces rejestracji.  Ale przed Rozpoczniemy tego procesu, który jest kilka rzeczy, podjęcie decyzji.

* **Telefon biurowy lub telefon komórkowy** — wybór między używaniem telefon biurowy telefon komórkowy.
* **Wywołanie telefonu komórkowego, tekst lub aplikacji mobilnej** — Jeśli przy użyciu telefonu komórkowego, wybór między odbieranie wywołania tekstu, przy użyciu aplikacji mobilnej.
* **Aplikacji mobilnej przy użyciu kodu weryfikacji lub powiadamiania** — Jeśli przy użyciu aplikacji mobilnej, wybór między otrzymujących powiadomienie, które należy odpowiedzieć lub kod weryfikacyjny.

> [!NOTE]
> Dział IT może wyłączyć co najmniej jeden z tych opcji.  Jeśli tak, te opcje nie będą dostępne podczas procesu rejestracji.  

Teraz, gdy użytkownik zdecydował się na metodę weryfikacji, jest czas do zarejestrowania. Użyj [środowiska instalacji](multi-factor-authentication-end-user-first-time.md) łącze do wprowadzenie do przewodnika dla procesu rejestracji.

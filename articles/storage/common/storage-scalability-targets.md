<span data-ttu-id="717b3-126">Każdy obiekt, który przechowuje dane, które są przechowywane w usłudze Azure Storage (obiekty BLOB, wiadomości, jednostki i plików) należy do partycji i jest identyfikowany przez klucz partycji.</span><span class="sxs-lookup"><span data-stu-id="717b3-126">Every object that holds data that is stored in Azure Storage (blobs, messages, entities, and files) belongs to a partition, and is identified by a partition key.</span></span> <span data-ttu-id="717b3-127">Partycja Określa, jak usługi Azure Storage obciążenia równoważy obiektów blob, wiadomości, jednostki i plików na serwerach, na potrzeby ruchu tych obiektów.</span><span class="sxs-lookup"><span data-stu-id="717b3-127">The partition determines how Azure Storage load balances blobs, messages, entities, and files across servers to meet the traffic needs of those objects.</span></span> <span data-ttu-id="717b3-128">Klucz partycji jest unikatowa i jest używana do lokalizowania obiektów blob, wiadomości lub jednostki.</span><span class="sxs-lookup"><span data-stu-id="717b3-128">The partition key is unique and is used to locate a blob, message, or entity.</span></span>

<span data-ttu-id="717b3-129">W tabeli [wartości docelowe skalowalności w przypadku kont magazynu standardowego](#standard-storage-accounts) wymieniono cele wydajności dla jednej partycji dla każdej usługi.</span><span class="sxs-lookup"><span data-stu-id="717b3-129">The table shown above in [Scalability Targets for Standard Storage Accounts](#standard-storage-accounts) lists the performance targets for a single partition for each service.</span></span>

<span data-ttu-id="717b3-130">Partycje wpływać równoważenia obciążenia i skalowalności dla każdej z usług magazynu w następujący sposób:</span><span class="sxs-lookup"><span data-stu-id="717b3-130">Partitions affect load balancing and scalability for each of the storage services in the following ways:</span></span>

* <span data-ttu-id="717b3-131">**Obiekty BLOB**: Klucz partycji dla obiekt blob jest nazwa konta, nazwa kontenera + nazwa obiektu blob.</span><span class="sxs-lookup"><span data-stu-id="717b3-131">**Blobs**: The partition key for a blob is account name + container name + blob name.</span></span> <span data-ttu-id="717b3-132">Oznacza to, że każdy obiekt blob może zawierać własnej partycji, jeśli obciążenie obiektu blob żąda jej.</span><span class="sxs-lookup"><span data-stu-id="717b3-132">This means that each blob can have its own partition if load on the blob demands it.</span></span> <span data-ttu-id="717b3-133">Obiekty BLOB mogą być rozproszone na wielu serwerach w celu skalowania w poziomie do nich dostęp, ale pojedynczego obiektu blob mogą być przekazywane tylko przez jeden serwer.</span><span class="sxs-lookup"><span data-stu-id="717b3-133">Blobs can be distributed across many servers in order to scale out access to them, but a single blob can only be served by a single server.</span></span> <span data-ttu-id="717b3-134">Obiekty BLOB można grupować logicznie w kontenerów obiektów blob, jednak niektóre powodują nie partycjonowania efekty z grupy.</span><span class="sxs-lookup"><span data-stu-id="717b3-134">While blobs can be logically grouped in blob containers, there are no partitioning implications from this grouping.</span></span>

<span data-ttu-id="717b3-143">Zaletą do grupowania zestawu jednostek w tabeli w jednej partycji jest możliwa do wykonywania operacji wsadowych atomic między jednostkami w tej samej partycji, ponieważ istnieje partycja na jednym serwerze.</span><span class="sxs-lookup"><span data-stu-id="717b3-143">A key advantage to grouping a set of entities in a table into a single partition is that it's possible to perform atomic batch operations across entities in the same partition, since a partition exists on a single server.</span></span> <span data-ttu-id="717b3-144">W związku z tym jeśli chcesz wykonać operacji wsadowych na grupy jednostek, należy wziąć pod uwagę ich grupowanie z tym samym kluczem partycji.</span><span class="sxs-lookup"><span data-stu-id="717b3-144">Therefore, if you wish to perform batch operations on a group of entities, consider grouping them with the same partition key.</span></span> 

<span data-ttu-id="717b3-145">Z drugiej strony, jednostek, które są w tej samej tabeli, ale ma różnych kluczach partycji mogą być o zrównoważonym obciążeniu na różnych serwerach, co umożliwia zwiększenie skalowalności.</span><span class="sxs-lookup"><span data-stu-id="717b3-145">On the other hand, entities that are in the same table but have different partition keys can be load balanced across different servers, making it possible to have greater scalability.</span></span>

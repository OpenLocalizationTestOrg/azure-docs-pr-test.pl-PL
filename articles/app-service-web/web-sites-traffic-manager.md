## <a name="load-balancing-methods"></a><span data-ttu-id="91396-109">Metody równoważenia obciążenia</span><span class="sxs-lookup"><span data-stu-id="91396-109">Load Balancing Methods</span></span>
<span data-ttu-id="91396-110">Trzy metody równoważenia obciążenia różnych korzysta z usługi Azure Traffic Manager.</span><span class="sxs-lookup"><span data-stu-id="91396-110">Azure Traffic Manager uses three different load balancing methods.</span></span> <span data-ttu-id="91396-111">Te są opisane na poniższej liście odnoszą się do aplikacji sieci web platformy Azure.</span><span class="sxs-lookup"><span data-stu-id="91396-111">These are described  in the following list as they pertain to Azure web apps.</span></span>

* <span data-ttu-id="91396-112">**Tryb failover**: Jeśli klony aplikacji sieci web znajdują się w różnych regionach, możesz ta metoda umożliwia skonfigurowanie jednej aplikacji sieci web do obsługi całego ruchu klientów sieci web i skonfiguruj inną aplikację sieci web w innym regionie do obsłużenia tego ruchu w przypadku pierwszej aplikacji sieci web staje się niedostępny.</span><span class="sxs-lookup"><span data-stu-id="91396-112">**Failover**: If you have web app clones in different regions, you can use this method to configure one web app to service all web client traffic, and configure another web app in a different region to service that traffic in case the first web app becomes unavailable.</span></span>

# <a name="how-do-azure-active-directory-admins-add-b2b-collaboration-users"></a>W jaki sposób administratorzy usługi Azure Active Directory dodają użytkowników we współpracy B2B?

Administratorzy globalni i Administratorzy ograniczone mogą używać [portalu Azure](https://portal.azure.com) z zaproszeniem dla użytkowników współpracy B2B w katalogu, do żadnej grupy lub do dowolnej aplikacji.

## <a name="admins-adding-guest-users-to-the-directory"></a>Administratorzy Dodawanie gości do katalogu
Dodawanie użytkowników współpracy B2B do katalogu jako administrator usługi Azure AD, jak pokazano w poniższym klipie wideo:

>[!VIDEO https://channel9.msdn.com/Blogs/Azure/Admin-adding-guests-for-B2B-collaboration-to-Azure-AD-tenant/Player]

Dodawanie użytkowników współpracy B2B do grupy jako administrator usługi Azure AD, jak pokazano w poniższym klipie wideo:

>[!VIDEO https://channel9.msdn.com/Blogs/Azure/admin-adding-guests-to-a-group/Player]


## <a name="admins-adding-guest-users-to-an-application"></a>Administratorzy Dodawanie gości do aplikacji

Dodaj użytkowników współpracy B2B do aplikacji jako administrator usługi Azure AD, jak pokazano w poniższym klipie wideo:

>[!VIDEO https://channel9.msdn.com/Blogs/Azure/admin-assign-b2b-to-apps/Player]

## <a name="admins-resending-invitations-to-guest-users"></a>Administratorzy ponowne wysyłanie zaproszeń do użytkowników gościa
Możesz przejść do strony profilu użytkownika współpracy B2B i ponownie wysłać zaproszenia dla żadnego użytkownika gościa nie zostały jeszcze zrealizowane:

![ponownie Wyślij zaproszenia](./media/active-directory-b2b-admin-add-users/resend-invitation.png)

> ! [UWAGA] W przypadku ponownego wysłania zaproszenia, zaproszenia są wysyłane z zalogowanego użytkownika do poszczególnych użytkowników, nawet jeśli oryginalny zaproszenia został wysłany do określonej aplikacji lub grupy.
